# 如何新增案件實績 — 業主維護指南

> 這份說明寫給不熟悉程式碼的業主看。  
> 只要按照步驟操作，不需要懂 HTML，就能把新案件放上官網。

---

## 一、新增一個案件，總共要做哪三件事？

| 步驟 | 要做的事 | 在哪裡操作 |
|------|---------|-----------|
| ① | 複製案件詳細頁樣板、填入案件內容 | GitHub 網頁 |
| ② | 把照片上傳到對應資料夾 | GitHub 網頁 |
| ③ | 在案件列表頁貼上一張卡片 | GitHub 網頁 |

三件事都完成後，等 **1～2 分鐘**，官網就會自動更新。

> 💡 **小提示：** 這三個步驟的所有內容（HTML 文字、照片說明、摘要），  
> 都可以把案件資料交給 AI（例如 ChatGPT 或 Claude）一次產生，  
> 業主只需要把 AI 產生的內容貼回 GitHub 即可。

---

## 二、詳細操作步驟

### 步驟 ① — 建立案件詳細頁

**1-1. 決定案件代號**

案件代號是這個案件的唯一識別碼，用英數字命名，例如：
- `proj-004`
- `proj-2025-robot-arm`
- `proj-taichung-conveyor`

**⚠️ 注意：代號一旦決定就不要改，因為照片路徑和連結都會用到它。**

---

**1-2. 在 GitHub 複製樣板**

1. 到您的 GitHub 專案頁面
2. 點進 `projects` 資料夾
3. 點選 `_template.html` 這個檔案
4. 右上角點「複製原始內容」圖示（或直接選取全部文字複製）

---

**1-3. 建立新檔案**

1. 回到 `projects` 資料夾
2. 點右上角 **「Add file」→「Create new file」**
3. 在檔名欄位輸入：`proj-004.html`（換成您的案件代號）
4. 把剛才複製的樣板內容貼進去
5. 搜尋所有 `【替換` 的地方，依序填入實際內容：

| 要替換的地方 | 填入什麼 |
|------------|---------|
| `【替換：案名】` | 案件名稱，例如：機械手臂組裝導入 |
| `【替換：客戶／產業】` | 例如：汽車零件製造業 ／ 台翔精機 |
| `【替換：設備類型】` | 例如：機械手臂整合應用 |
| `【替換：完成日期（選填）】` | 例如：2025 年 Q2，或直接刪除這行 |
| `【替換：案件代號】` | 換成您的代號，例如：proj-004 |
| 專案背景、解決方案、成果效益 | 依實際狀況撰寫（見下方說明） |
| `【替換：案名】施工現場 1` 等 alt 文字 | 描述照片內容即可 |

**文字建議：**
- **專案背景**：客戶原本遇到什麼問題？（2～3 句）
- **解決方案**：我們導入了什麼設備、怎麼整合？（2～3 句）
- **成果效益**：具體數字最有說服力，例如「產能提升 40%」、「節省 2 名人力」

6. 填完後，頁面往下滑，點 **「Commit changes」** → 再點綠色 **「Commit changes」** 確認

---

### 步驟 ② — 上傳照片

**2-1. 建立照片資料夾並上傳**

1. 從 GitHub 專案根目錄，點進 `images` → `projects`
2. 點右上角 **「Add file」→「Upload files」**
3. 把照片拖拉進去
4. 在「Commit changes」之前，**先在頁面最上方的路徑欄**（顯示 `projects/`）  
   的後面輸入：`proj-004/`（換成您的代號）  
   這樣照片就會建在正確的子資料夾裡

**照片命名規則：**
- 用 `01.jpg`、`02.jpg`、`03.jpg` … 依序命名
- **不要用中文檔名**，不要有空格
- 建議每張照片先壓縮到 500KB 以下（可用免費工具 squoosh.app）

**2-2. 更新 HTML 裡的照片路徑**

上傳完照片後，回到剛才建立的 `proj-004.html`，找到相簿區段：

```html
<img src="https://picsum.photos/seed/.../800/600" alt="...">
```

把 `https://picsum.photos/...` 這段改成實際路徑，例如：

```html
<img src="../images/projects/proj-004/01.jpg" alt="機械手臂組裝現場">
```

有幾張照片就改幾行。如果照片比預設的 4 張更多，可以複製整個 `<div class="gallery-item">...</div>` 區塊往下貼，更改檔名即可。

---

### 步驟 ③ — 在案件列表頁貼卡片

1. 從根目錄點開 `projects.html`
2. 點右上角鉛筆圖示（Edit this file）進入編輯模式
3. 找到下面這段文字（在檔案裡搜尋 `日後新增案件貼在此處`）：

```
<!-- 日後新增案件貼在此處 ↑ -->
```

4. 在這行**上方**貼入以下卡片段落，替換 `【】` 內的內容：

```html
<a class="project-card" href="projects/proj-004.html">
  <div class="project-card-img-wrap">
    <img src="images/projects/proj-004/01.jpg" alt="機械手臂組裝導入" loading="lazy">
  </div>
  <div class="project-card-body">
    <h3>機械手臂組裝導入</h3>
    <p class="card-tag">汽車零件製造業 ／ 台翔精機</p>
    <p class="card-summary">導入協作型機器人取代人工鎖螺絲工站，節省 2 名人力，組裝一致性大幅提升。</p>
    <span class="card-arrow">查看詳情 →</span>
  </div>
</a>
```

5. 點 **「Commit changes」** 儲存

## 三、每個欄位說明

| 欄位 | 說明 | 範例 |
|------|------|------|
| 案名（h3）| 短而有力，讓人一眼看出做了什麼 | 協作機器人鎖螺絲工站導入 |
| 客戶／產業 | 產業類別 ＋ 公司簡稱（若客戶不便公開公司名，只寫產業也可以） | 汽車零件製造業 ／ 台翔精機 |
| 設備類型 | 與產品頁對應的類別名稱 | 機械手臂整合應用 |
| 一句話摘要 | 說明解決了什麼、數字效益（約 30 字內） | 節省 2 名人力，組裝節拍縮短 40% |
| 縮圖（列表頁）| 建議使用第一張照片 01.jpg | `images/projects/proj-004/01.jpg` |

## 四、上傳後網站多久更新？

GitHub Pages 在您 **Commit（儲存）** 後，通常 **1～2 分鐘**內就會自動重新發布。  
若超過 5 分鐘還沒更新，請到 GitHub 專案頁面點 **「Actions」** 標籤，確認部署流程是否有錯誤。

---

## 五、如何開啟 GitHub Pages（第一次部署）

1. 到 GitHub 專案頁面，點上方 **「Settings」**
2. 左側選單點 **「Pages」**
3. 在「Branch」下拉選單選 **`main`**，資料夾選 **`/ (root)`**
4. 點 **「Save」**
5. 等 1～2 分鐘，頁面會顯示：  
   `Your site is live at https://您的帳號.github.io/專案名稱/`

---

## 六、如何綁定自訂網域（選用）

若業主已購買網域（例如 `www.your-company.com.tw`）：

1. 到網域商後台，新增一筆 **CNAME 記錄**：  
   - 名稱：`www`  
   - 值：`您的GitHub帳號.github.io`
2. 回到 GitHub → Settings → Pages → 「Custom domain」  
   輸入 `www.your-company.com.tw` 並儲存
3. 勾選 **「Enforce HTTPS」**（建議開啟）
4. DNS 生效時間約 10 分鐘～24 小時

## 八、常見問題

**Q：照片用中文檔名會怎樣？**  
A：可能在某些瀏覽器或作業系統上顯示異常。請一律用英數字命名（01.jpg、02.jpg）。

**Q：可以不放照片嗎？**  
A：可以，但建議至少放 1 張。把 `<div class="gallery-item">...</div>` 整段刪除即可，燈箱功能會自動停用。

**Q：案件順序可以調整嗎？**  
A：可以。在 `projects.html` 裡，最上方的卡片會顯示在最前面，把整段卡片 HTML 上下剪貼即可調整順序。首頁的精選案件則要到 `index.html` 手動更新。

**Q：想修改公司名稱、電話等基本資料，要改幾個地方？**  
A：每個頁面的頁首和頁尾都有一份。全站共有 8 個 HTML 檔，每個都要改。  
建議用 GitHub 的「搜尋」功能（在所有檔案裡搜尋舊文字），或一次交給 AI 批次替換。
